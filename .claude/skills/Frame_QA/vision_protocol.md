# Vision 기반 UI 이상 탐지 프로토콜

> **스크린샷을 Claude Vision으로 분석하여 가시적 UI 이상을 구조적으로 탐지한다.**
> Phase 2의 스크린샷 검증 단계에서 이 프로토콜을 적용한다.

## UI 이상 카테고리 분류표

| # | 카테고리 | 탐지 대상 | 심각도 기본값 |
|---|---------|----------|-------------|
| 1 | **Layout** | 정렬 틀어짐, UI 요소 오버랩, 잘림(clipping), 비정상 간격, 화면 밖 UI | Critical |
| 2 | **Text** | 텍스트 깨짐, 넘침(overflow), 폰트 불일치, 빈 텍스트, 타입명 노출 (`D_String`, `System.Object` 등) | Critical |
| 3 | **Asset** | 핑크/마젠타 사각형 (missing texture), 빈 이미지 슬롯, Addressable 누락, 깨진 스프라이트 | Critical |
| 4 | **Color/Contrast** | 배경과 텍스트 구분 불가, 테마 불일치, 비정상 투명도 | Warning |
| 5 | **Animation** | 프레임 멈춤, 미완성 트랜지션, 깜빡임, 위치 점프 | Warning |
| 6 | **Z-Order** | UI 요소 가림, 팝업 뒤에 인터랙션 가능한 요소 노출, sortingOrder 역전 | Critical |
| 7 | **State** | 비활성이어야 할 UI 활성, 빈 리스트/빈 컨테이너, "0/0" 같은 초기화 안된 값, 로딩 스피너 잔존 | Warning |

## Vision 분석 프롬프트 템플릿

### 템플릿 1: 일반 UI 검증

스크린샷 캡처 후 `Read` 도구로 이미지를 읽고, 아래 관점으로 분석한다:

```
이 게임 UI 스크린샷을 분석하여 다음 카테고리별 이상을 보고하라:

1. Layout: 정렬 틀어짐, 오버랩, 잘림(clipping), 비정상 간격이 있는가?
   - **특히 주의**: UI 요소가 부모 영역을 벗어나 잘려 보이는가? 화면 경계 밖으로 밀려난 UI가 있는가?
   - 텍스트나 이미지가 컨테이너보다 커서 일부만 보이는 현상이 있는가?
2. Text: 텍스트 깨짐, 넘침, 빈 텍스트, 타입명 노출(D_String 등)이 있는가?
3. Asset: 핑크/마젠타 영역, 빈 이미지, 깨진 스프라이트가 있는가?
4. Color/Contrast: 읽기 어려운 텍스트, 테마 불일치가 있는가?
5. Z-Order (레이어 순서): 가려진 UI, 팝업 뒤 요소 노출이 있는가?
   - **특히 주의**: 팝업이 다른 UI 뒤에 깔려 있거나, 상위 레이어여야 할 요소가 하위에 표시되는가?
   - 모달 팝업 뒤로 클릭 가능한 버튼이 보이는가?
   - UI 요소 간 의도치 않은 겹침(오버랩)이 있는가?
6. State: 초기화 안된 값("0/0" 등), 비활성이어야 할 UI가 보이는가?

각 발견 항목에 대해 [카테고리] [위치 설명] [심각도: Critical/Warning/Info] 형식으로 보고하라.
발견 없으면 "VISION_CLEAR" 로 보고.
```

### 템플릿 2: 전투 UI 특화

```
이 전투 씬 스크린샷을 분석하여 다음을 확인하라:

1. HP 바: PC와 NPC의 HP 바가 올바르게 표시되는가? "0/0" 등 초기화 안된 값은 없는가?
2. 카드 UI: 핸드의 카드가 정상적으로 렌더링되는가? 카드 이미지/텍스트가 깨지지 않았는가?
3. 오더체인: 카드 배치 슬롯이 올바르게 표시되는가?
4. 페이즈 표시: 현재 전투 페이즈가 UI에 반영되는가?
5. 버튼 상태: Run 버튼 등 액션 버튼이 적절한 활성/비활성 상태인가?
6. 미니맵/엔티티 UI: NPC 아이콘, 체력 표시가 정상인가?
7. 이펙트/애니메이션: 전투 이펙트가 UI를 가리거나 잔존하지 않는가?

각 발견 항목: [카테고리] [위치] [심각도] [설명]
```

### 템플릿 3: 비교 분석 (전/후 또는 기대/실제)

```
두 스크린샷을 비교 분석하라:
- 이전 (또는 기대 상태): [첫 번째 스크린샷]
- 이후 (또는 실제 상태): [두 번째 스크린샷]

다음을 확인하라:
1. 의도된 변화: 액션에 의해 변경되어야 할 UI 요소가 실제로 변경되었는가?
2. 의도치 않은 변화: 변경되지 않아야 할 UI 요소가 변경되었는가?
3. 새로 나타난 이상: 이전에는 없던 UI 이상이 새로 발생했는가?
4. 해소된 이상: 이전에 있던 이상이 해소되었는가?

보고: [변화 유형: 의도됨/의도치않음/신규이상/해소] [위치] [설명]
```

## 참조 스크린샷 시스템

```
_qa/
  screenshots/
    reference/       -- 정상 상태 기준 스크린샷 (첫 실행 시 생성, 수동 승인 후 유지)
    current/         -- 현재 실행 캡처 (Phase 3에서 정리)
    issues/          -- 이상 발견 시 보존 ({날짜}_{시나리오}_{step}.png)
```

**운용 규칙:**
- **첫 실행**: `reference/`가 비어있으면 현재 캡처를 reference로 복사. 보고서에 "초기 reference 생성됨" 명시.
- **이후 실행**: `current/` 캡처를 `reference/`와 비교 분석 (템플릿 3 적용)
- **reference 갱신**: 버그 수정 후 해당 시점의 reference를 수동으로 갱신 (자동 덮어쓰기 금지)
- **이상 보존**: Vision 분석에서 Warning 이상 이슈 발견 시 `issues/`에 복사 보존

## Vision 이슈 심각도 자동 분류 기준

| 심각도 | 조건 |
|--------|------|
| **Critical** | 핑크/마젠타 텍스처, 타입명 노출, HP "0/0", UI 완전 가림, 빈 화면 |
| **Warning** | 텍스트 넘침, 경미한 정렬 틀어짐, 색상 대비 부족, 애니메이션 멈춤 |
| **Info** | 미세한 간격 불일치, 폰트 크기 차이, 비기능적 시각 이상 |
